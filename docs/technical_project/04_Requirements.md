Ниже рассмотрены различные требования, предъявляемые к текущему решению.

### Бизнес-требования

**БТ-01.** Система должна работать в режиме "`здесь и сейчас`".  
Пользователь стоит перед экраном регистрации и не готов ждать анализ дольше 30-60 секунд.

**БТ-02.** Пользователь не должен обладать юридическими знаниями для понимания отчёта. Все риски должны быть переведены на простой язык (Plain Russian).

**БТ-03.** Сервис должен снимать с себя ответственность за принятие решения пользователем (наличие строгого Дисклеймер).

**БТ-04.** Возможность обработки документов как в виде файлов (.pdf, .txt), так и в виде простого текста (copy-paste).

**БТ-05.** Оптимизация затрачиваемых средств на сопровождение решения и, как следствие, снижение затрат пользователей на использование сервиса.

**БТ-06.** Использование преимущественно доступных средств с возможностью их монетизации.

### Нефункциональные требования

#### Масштабируемость

- Система должна поддерживать до 1000 одновременных пользователей
- Векторная база должна масштабироваться горизонтально
- n8n workflows должны поддерживать параллельную обработку

#### Производительность

- Время анализа документа: не более 60 секунд
- Время отклика бота: не более 2 секунд
- Время загрузки файла: зависит от размера, но не более 30 секунд для 50 МБ

#### Удобство использования

- Интерфейс бота должен быть интуитивно понятным
- Все сообщения на русском языке (Plain Russian)
- Поддержка мобильных устройств
- Минимум кликов для выполнения анализа

#### Надёжность

- Uptime: 99% в дневное время (8:00-19:00 МСК)
- Автоматическое восстановление после сбоев
- Graceful degradation при недоступности компонентов

#### Безопасность

- Транзитная обработка данных (без сохранения на диск)
- Маскирование PII перед передачей в LLM
- Шифрование данных при передаче (HTTPS/TLS)
- Логирование без содержимого документов

#### Защищённость

- Защита от DDoS атак
- Rate limiting для API
- Защищённый доступ к административной панели (2FA)
- Регулярные резервные копии базы знаний

#### Совместимость

- Поддержка форматов: PDF, TXT, изображения (PNG, JPG)
- Работа во всех популярных Telegram-клиентах
- Кроссплатформенность (iOS, Android, Desktop)

#### Переносимость

- Возможность развертывания на различных облачных платформах
- Контейнеризация компонентов (Docker)
- Использование стандартных протоколов и форматов данных
