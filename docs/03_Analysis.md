## 3 Анализ предметной области

### 3.1 Анализ альтернатив

В настоящий момент на рынка существует ряд решений, решающих подобные бизнес-цели.

**[ИЗОБРАЖЕНИЕ: Таблица сравнения альтернатив - Grammarly, LawGeex, DocuBank, Zuva, TermsFeed]**

Таблица включает следующие решения:

1. **Grammarly** (https://www.grammarly.com/)
   - Функциональные возможности: AI-платформа для проверки и улучшения текстов на английском языке
   - Минусы: Отсутствие поддержки русского языка, блокировка в России, несовместимость с российскими юридическими документами

2. **LawGeex** (https://www.lawgeex.com/)
   - Функциональные возможности: AI-система для автоматизированного анализа и редактирования контрактов
   - Минусы: Ориентация на англосаксонскую правовую систему, отсутствие русскоязычного интерфейса, ограниченная применимость

3. **DocuBank** (https://www.docubank.com/)
   - Функциональные возможности: Электронная система хранения и доступа к важным документам
   - Минусы: Отсутствие локализации, нет функции анализа, несовместимость с российским документооборотом

4. **Zuva** (https://zuva.ai/)
   - Функциональные возможности: Специализированная B2B-платформа для анализа контрактов с использованием AI
   - Минусы: Англоязычная база знаний, B2B-модель для крупных организаций, незнание российского законодательства

5. **TermsFeed** (https://www.termsfeed.com/)
   - Функциональные возможности: Генератор юридических документов для веб-сайтов и приложений
   - Минусы: Генерация по западным стандартам, отсутствие русскоязычных шаблонов, нет функции анализа

### 3.2 Типы документов

Основные типы документов представлены в таблице:

**[ИЗОБРАЖЕНИЕ: Таблица типов документов с колонками: №, Тип документа, Где встречается, Объём, Особенности на которые нужно обратить внимание, Риски]**

Таблица включает 17 типов документов:

1. **Terms of Service (ToS) / Условия использования**
   - Где встречается: практически все онлайн-сервисы
   - Объём: 3000-8000 слов
   - Особенности: Одностороннее право на изменение правил, право на сбор неограниченных данных, отказ от гарантий, отказ от возврата денег, сложность отмены подписки

2. **Privacy Policy / Политика конфиденциальности**
   - Где встречается: все сервисы с обработкой персональных данных
   - Объём: 2000-6000 слов
   - Риски: для РФ → ВААП, для ЕС → GDPR штрафы

3. **User Agreement / Пользовательское соглашение**
   - Где встречается: образовательные платформы, маркетплейсы, социальные сети
   - Объём: 4000-10000 слов
   - Риски: риск потери доступа к аккаунту

4. **End User License Agreement (EULA)**
   - Где встречается: ПО, видеоигры, мобильные приложения
   - Объём: 2000-5000 слов

5. **Service Level Agreement (SLA)**
   - Где встречается: облачные сервисы, хостинг, B2B решения
   - Объём: 1000-3000 слов

6. **Оферта**
   - Где встречается: E-commerce, услуги, подписки
   - Объём: 1500-4000 слов
   - Риски: финансовые потери

7. **Cookie Policy / Политика использования куков**
   - Где встречается: веб-сайты, онлайн-сервисы
   - Объём: 800-2000 слов

8. **Code of Conduct / Кодекс поведения**
   - Где встречается: образовательные платформы (Coursera, Udemy, LMS системы)
   - Риски: потеря контента, сертификата

9. **Payment Terms**
   - Где встречается: платёжные системы (Stripe, PayPal, Яндекс.Касса, 2Checkout)
   - Объём: 3000-8000 слов
   - Риски: блокировка денег

10. **Data Processing Agreement (DPA)**
    - Где встречается: облачные сервисы (AWS, Google Cloud, Yandex Cloud)
    - Объём: 2000-6000 слов
    - Риски: утечка данных клиента

11. **Seller Agreement**
    - Где встречается: маркетплейсы (eBay, Amazon, Avito, Ozon)
    - Риски: потеря дохода

12. **Content License**
    - Где встречается: видеохостинги
    - Риски: потеря авторства

13. **Platform Terms**
    - Где встречается: соцсети/мессенджеры
    - Риски: потеря доступа и приватности

14. **Подписки SaaS / Subscription Terms**
    - Где встречается: Adobe, Figma, Notion, Slack
    - Риски: регулярные платежи

15. **API Terms of Service**
    - Где встречается: собственный API, интеграция с банками, платёжными сервисами

16. **Medical Disclaimer**
    - Где встречается: фитнес-приложения, здоровье-сервисы
    - Риски: правовая защита слаба

17. **Policy Terms / Условия полиса**
    - Где встречается: страховые компании
    - Риски: потеря денег при страховом случае

Пользователь самостоятельно несет полную ответственность за принятое решение о заключении договора.

4. **Пригодность для конкретного случая:**  
   Сервис создан для массовых сценариев (оферты онлайн-школ, политики конфиденциальности интернет-сервисов). Он может быть неполон для специализированных контрактов (например, высокостоимостные B2B соглашения, договоры в судебных спорах). Рекомендуется проверить анализ с юристом для таких случаев.

6. **Конфиденциальность загруженных данных:**  
   Загруженные документы обрабатываются в оперативной памяти и удаляются сразу после анализа. Данные пользователя не сохраняются в базах данных (исключение: логи технических ошибок с маскированием PII на срок 30 дней). 8. **Ограничение ответственности (Liability Limitation):**
